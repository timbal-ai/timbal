Knowledge Bases

Store, organize, and query structured data with AI-optimized storage and semantic search.

Platform Feature

Knowledge Bases are currently only available for Timbal Platform users. This functionality requires internal authentication tokens and endpoints that are not available in the open source distribution.

Knowledge Bases in Timbal provide a powerful way to store, organize, and query structured data. They allow you to create tables, manage indexes, import data from various sources, and perform SQL queries to extract insights from your data.

What is a Knowledge Base?​

A Knowledge Base is like having your own private database that's specifically designed for AI applications. Think of it as a smart storage system that not only holds your data but also understands it in ways that make it perfect for AI agents to work with.

Why Use Knowledge Bases?​

Traditional databases are great for storing data, but they're not optimized for AI workflows. Knowledge Bases bridge this gap by providing:

  • AI-Optimized Storage: Tables designed with AI use cases in mind
  • Semantic Understanding: Built-in embedding generation for finding similar content
  • Flexible Schema: Easy to modify and adapt as your data needs change
  • Performance Optimization: Intelligent indexing for fast AI-powered queries
  • Integration Ready: Seamlessly connects with Timbal agents and flows

Common Use Cases​

Knowledge Bases are perfect for:

  • Document Management: Store and search through large document collections
  • Customer Data: Organize customer information for personalized AI interactions
  • Product Catalogs: Create searchable product databases
  • Research Data: Store and analyze research papers, articles, or datasets
  • Knowledge Graphs: Build interconnected knowledge networks
  • Training Data: Store examples and training data for AI models

How It Works​

Knowledge Bases use PostgreSQL under the hood but add AI-specific features:

  1. Table Creation: Define your data structure with custom schemas
  2. Data Import: Load data from CSV files or add records programmatically
  3. Indexing: Create optimized indexes for fast queries
  4. Embedding Generation: Automatically create vector embeddings for semantic search
  5. Querying: Use SQL or semantic search to retrieve relevant information

Getting Started​

Platform Access Required

To use Knowledge Base functionality, you need access to the Timbal Platform with valid organization credentials.

Table Management​

Tables are the foundation of your Knowledge Base. Think of them as organized containers for your data, where each table represents a specific type of information you want to store and query.

Understanding Table Structure​

Before creating tables, it's important to understand the key components:

  • Columns: Define what data each row contains (like fields in a form)
  • Data Types: Specify how the data should be stored (text, numbers, dates, etc.)
  • Constraints: Set rules for your data (required fields, unique values, etc.)
  • Comments: Add descriptions to help understand your schema

Creating Tables​

You can create tables with custom schemas using the create_table function. This is like designing a blueprint for your data:

from timbal.platform.kbs import create_table
from pydantic import BaseModel, Field


class Column(BaseModel):
  name: str = Field(..., min_length=1, max_length=128)
  data_type: str
  default_value: Optional[str] = None
  is_nullable: bool
  is_unique: bool
  comment: Optional[str] = Field(default=None, min_length=1, max_length=256)


# Define table columns
columns = [
  Column(
      name="id",
      data_type="SERIAL",
      is_nullable=False,
      is_unique=True,
      comment="Primary key"
  ),
  Column(
      name="title",
      data_type="VARCHAR(255)",
      is_nullable=False,
      is_unique=False,
      comment="Document title"
  ),
  Column(
      name="content",
      data_type="TEXT",
      is_nullable=True,
      is_unique=False,
      comment="Document content"
  ),
  Column(
      name="created_at",
      data_type="TIMESTAMP",
      default_value="CURRENT_TIMESTAMP",
      is_nullable=False,
      is_unique=False
  )
]


# Create the table
await create_table(
  table_name="documents",
  columns=columns
)

Listing Tables​

Once you've created tables, you can see what's available in your Knowledge Base. This is useful for understanding your data structure and planning queries:

from timbal.platform.kbs import list_tables


tables = await list_tables()
print(f"Available tables: {tables}")

Getting Table Information​

Understanding your table structure is crucial for effective data management. You can get different levels of detail about your tables:

from timbal.platform.kbs import get_table, describe_table, get_table_definition


# Get basic table info
table_info = await get_table(table_name="documents")


# Get detailed column descriptions
table_description = await describe_table(
  table_name="documents",
  columns=["title", "content"]  # Optional: specify columns to describe
)


# Get table definition (DDL)
table_definition = await get_table_definition(table_name="documents")

Deleting Tables​

Sometimes you need to clean up your Knowledge Base by removing tables that are no longer needed. Be careful with this operation as it permanently removes data:

from timbal.platform.kbs import delete_table


# Delete table with cascade (removes indexes too)
await delete_table(
  table_name="documents",
  cascade=True
)

Data Import​

Getting data into your Knowledge Base is a crucial step. Timbal provides multiple ways to import data, making it easy to work with different data sources and formats.

Understanding Data Import​

Data import is the process of loading your existing data into Knowledge Base tables. This could be:

  • CSV files from spreadsheets or exports
  • Programmatic data from your applications
  • Batch imports for large datasets
  • Real-time updates as new data becomes available

Importing CSV Files​

CSV files are one of the most common data formats. They're easy to create from spreadsheets and are perfect for bulk data imports:

from timbal.platform.kbs import import_csv


# Import CSV file
await import_csv(
  table_name="documents",
  csv_path="/path/to/your/data.csv"
)

Importing Records​

Sometimes you need to add data directly from your Python code. This is perfect for:

  • Real-time data from APIs or sensors
  • Processed data that you've transformed or cleaned
  • Small batches of data that don't warrant a CSV file
  • Dynamic data that changes frequently
from timbal.platform.kbs import import_records


# Prepare records
records = [
  {
      "title": "Introduction to AI",
      "content": "Artificial Intelligence is a field of computer science...",
      "created_at": "2024-01-15T10:00:00Z"
  },
  {
      "title": "Machine Learning Basics",
      "content": "Machine Learning is a subset of AI...",
      "created_at": "2024-01-16T14:30:00Z"
  }
]


# Import records
await import_records(
  table_name="documents",
  records=records
)

Index Management​

Indexes are like the index at the back of a book - they help you find information quickly without having to search through everything. In Knowledge Bases, indexes dramatically improve query performance.

Understanding Indexes​

Think of an index as a pre-computed lookup table that tells the database where to find specific data. Without indexes, the database would have to scan every row to find what you're looking for (like reading every page of a book to find a topic).

Why Indexes Matter​

  • Speed: Queries run much faster with proper indexes
  • Efficiency: Reduces the computational load on your database
  • Scalability: Performance stays good even as your data grows
  • Cost: Faster queries mean lower computational costs

Creating Indexes​

Optimize query performance by creating indexes. Choose the right index type for your use case:

from timbal.platform.kbs import create_index


# Create a B-tree index on title column
await create_index(
  table_name="documents",
  index_name="idx_documents_title",
  index_columns=["title"],
  index_type="btree",
  index_unique=False
)


# Create a unique hash index on id
await create_index(
  table_name="documents",
  index_name="idx_documents_id_unique",
  index_columns=["id"],
  index_type="hash",
  index_unique=True
)

Listing Indexes​

Keeping track of your indexes helps you understand your database performance and plan optimizations:

from timbal.platform.kbs import list_indexes


indexes = await list_indexes()
for index in indexes:
  print(f"Index: {index.name} on table {index.table}")
  print(f"Columns: {index.columns}")
  print(f"Type: {index.type}")
  print(f"Unique: {index.is_unique}")

Deleting Indexes​

Sometimes indexes become unnecessary or even counterproductive. You might want to remove them when:

  • Performance changes: The index is no longer helping
  • Schema changes: The indexed columns are modified or removed
  • Storage optimization: Removing unused indexes to save space
  • Maintenance: Cleaning up after testing different index strategies
from timbal.platform.kbs import delete_index


await delete_index(index_name="idx_documents_title")

Querying Data​

Querying is where the real power of Knowledge Bases comes to life. You can ask complex questions about your data and get precise answers quickly.

Understanding Queries​

Queries are like asking questions to your data. Instead of manually searching through spreadsheets or files, you can write a query that tells the database exactly what you want to know.

Types of Queries​

  • Simple queries: Find specific records or basic information
  • Complex queries: Join multiple tables, aggregate data, or perform calculations
  • Analytical queries: Discover patterns, trends, or insights in your data
  • Real-time queries: Get up-to-date information as it changes

SQL Queries​

SQL (Structured Query Language) is the standard way to interact with databases. It's powerful, flexible, and allows you to express complex data requirements:

from timbal.platform.kbs import query


# Simple SELECT query
results = await query(
  sql_query="SELECT * FROM documents WHERE title LIKE '%AI%'"
)


# Complex query with JOIN
results = await query(
  sql_query="""
  SELECT d.title, d.content, COUNT(*) as word_count
  FROM documents d
  WHERE d.created_at >= '2024-01-01'
  GROUP BY d.id, d.title, d.content
  ORDER BY word_count DESC
  LIMIT 10
  """
)


# Aggregation query
results = await query(
  sql_query="""
  SELECT 
      DATE(created_at) as date,
      COUNT(*) as documents_count,
      AVG(LENGTH(content)) as avg_content_length
  FROM documents
  GROUP BY DATE(created_at)
  ORDER BY date DESC
  """
)

Embeddings and Semantic Search​

Embeddings are what make Knowledge Bases truly "smart." They transform your text data into numerical representations that capture meaning, enabling semantic search and AI-powered analysis.

Understanding Embeddings​

Think of embeddings as a way to convert words and sentences into numbers that represent their meaning. Similar concepts get similar numbers, making it possible to find related content even when the exact words don't match.

Why Embeddings Matter​

  • Semantic Search: Find content based on meaning, not just keywords
  • Similarity Detection: Discover related documents or concepts
  • AI Integration: Enable machine learning models to understand your data
  • Natural Language: Query your data using natural language

Creating Embeddings​

Generate embeddings for semantic search capabilities. This process converts your text data into vector representations:

from timbal.platform.kbs import create_embeddings


# Create embeddings for content column
await create_embeddings(
  table_name="documents",
  model="text-embedding-3-small",
  source_column="content"
)

Index Types​

Choosing the right index type is crucial for optimal performance. Each type is designed for specific use cases:

B-tree Indexes (btree)​

  • Best for: Equality and range queries, sorting
  • Use cases: Primary keys, foreign keys, date ranges, alphabetical searches
  • Example: Finding users by email, products by price range
  • Performance: Excellent for most common query patterns

Hash Indexes (hash)​

  • Best for: Exact equality matches only
  • Use cases: Unique identifiers, exact lookups
  • Example: Finding a user by their unique ID
  • Performance: Very fast for exact matches, but no range queries

GIN Indexes (gin)​

  • Best for: Composite values, arrays, full-text search
  • Use cases: JSON data, array fields, text search
  • Example: Searching through tags, JSON properties, or document content
  • Performance: Excellent for complex data types

GiST Indexes (gist)​

  • Best for: Geometric data, full-text search, custom data types
  • Use cases: Geographic data, text search with ranking
  • Example: Finding locations within a radius, ranked text search
  • Performance: Good for specialized data types

BRIN Indexes (brin)​

  • Best for: Large tables with natural ordering
  • Use cases: Time-series data, sequential data
  • Example: Log data, sensor readings, time-stamped records
  • Performance: Very space-efficient for large datasets

Knowledge Bases transform how you work with data in AI applications. They're not just databases - they're intelligent storage systems designed specifically for AI workflows.
